# Quickstart — Chat UI (002-chat-ui)

Run, develop, and test the Chat UI. `docker compose up` is the only
supported start path.

---

## Prerequisites

- Docker Desktop (or any Docker engine) + Docker Compose v2.
- A populated `.env` at the repo root. Copy from `.env.example` and set
  `APP_SHARED_TOKEN` (free-form string) plus one of:
  - `ANTHROPIC_API_KEY` (if `LLM_PROVIDER=anthropic`), **or**
  - `OPENAI_API_KEY` (if `LLM_PROVIDER=openai`).
- No host-side Node, Python, or `npm` is required. The toolchain lives
  in the containers.

---

## Run (production-style: SPA served by FastAPI on `:8000`)

```bash
docker compose up --build
```

Open <http://localhost:8000/>. Expect:

1. The chat UI loads. Upload surface is visible.
2. Drag a PDF or DOCX in (≤ 25 MB). Watch the progress indicator until
   "ready".
3. Type a question. Press Enter. First token visible within ~2 s.
4. Refresh the tab. Transcript rehydrates from
   `GET /history/{session_id}` (same tab only).
5. Click "New Session". Transcript clears; server-side session purged
   via `POST /session/end`.

Stop: `docker compose down`.

---

## Develop the SPA with hot-reload

```bash
docker compose --profile dev up
```

- Backend at <http://localhost:8000/> (no SPA mounted in dev container
  because the dev `frontend-dev` service serves the SPA itself).
- SPA dev server at <http://localhost:5173/> with HMR. API calls
  proxied to the backend over the compose network.
- Edit any file in `frontend/src/**` — Vite reloads.

Stop: `docker compose --profile dev down`.

## Tests

All tests run inside the production image; no host toolchain.

### Backend tests (existing)

```bash
docker compose run --rm app pytest
```

This run now also covers the **new** unit test for the same-origin
auth bypass in `src/api/deps.py`.

### SPA unit, contract, and streaming integration tests

```bash
docker compose --profile dev run --rm frontend-dev npm test -- --run
```

Single Vitest 4 invocation covers:

- Unit (reducer, parser wiring, persistence, component behavior).
- Contract (MSW 2 fakes for every backend route the SPA uses; typed
  client generated by `openapi-typescript` 7).
- Streaming integration: drives the production `api/ask.ts` against an
  MSW 2 streaming `Response`, asserts ≥ 2 incremental `token`
  dispatches before `done` (consumer-side streaming gate), and asserts
  `AbortController.abort()` mid-stream marks the trailing turn as
  `stopped`. **No headless browser, no Playwright.**

## What to look at if something is wrong

| Symptom | Likely cause | Where to look |
|---------|--------------|---------------|
| SPA loads but every API call returns 401 | Backend not honoring same-origin bypass | `src/api/deps.py` — the dependency must inspect `Origin`/`Referer` against the served host |
| First token takes > 5 s | LLM provider slow or unreachable | Server logs (`docker compose logs app`), check `LLM_PROVIDER` + matching API key in `.env` |
| Streaming text appears all at once | SSE buffered (proxy or middleware) | Confirm no compression middleware (e.g., `GZipMiddleware`) wraps `/ask` |
| Refresh wipes the transcript | `sessionStorage` key missing or `/history/{id}` returned 404 | Browser devtools → Application → Session Storage; backend log for `history` route |
| Unauthorized error from `curl` against `/upload` | Expected — non-browser callers still need the bearer token | Pass `-H "Authorization: Bearer $APP_SHARED_TOKEN"` |
| Big bundle warning at build | Tailwind 4 source detection broke or a heavy dep slipped in | Confirm `@tailwindcss/vite` plugin registered in `vite.config.ts` and `@import "tailwindcss";` is at the top of `src/styles/tailwind.css`; `npm run analyze` |
